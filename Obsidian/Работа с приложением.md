## Базированная работа 

1. Для обновления ветки dev, просто ищем окно с названием "Source control: changes" (обычно справа, под файлами), пишем сообщение для git, проверяем, какие изменения вносятся ==**ВНИМАТЕЛЬНО**==, нажимаем на кнопку "Commit", соглашаемся, и дальше синхроним с GitHub. Ну и обновляем контейнер)
2. Можно поглядеть плагин GitLens, удобно показывает все коммиты
3. Работаем только из папок, они разные для веток и соответствуют им, переключаться внутри одной папки между ветками нельзя!
4. Подключение идет по SSH, оно уже настроено ![[{867C0511-A5AA-49A0-A831-A0CEF791571F}.png]] (компутер слева) 
5. Для веток также есть две разные БД ![[{BFCF7FC2-A3BF-439C-AF86-39C1F684227B}.png]] (потыкайся, там можно выводить все данные таблицы по нажатию, запрос пишется прямо в VS Code )
## Как выгружать коммиты из ветки dev в ветку main

1. В терминал на ветке **main** нужно ввести команды по одному
	git fetch origin *(обновление ссылок на удалённые ветки)*
   
	git checkout main *(переход в main и подтягивание актуального)*
	git pull origin main *(переход в main и подтягивание актуального)*
   
	git log --oneline origin/dev -n 20 *(получение 20 последних коммитов на ветке dev)*
   
	git cherry-pick hash1 hash2 *(перенос коммитов, пишем без запятых)*
   
	git push origin main *(пушим коммиты на удаленную ветку)*

Также хэши можно получить из самого GitHub, надо открыть коммиты, выбрать ветку dev и у каждого коммита будет написан хэш.
